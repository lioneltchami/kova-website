# kova status

Display the current state of the most recent build: task statuses, timing, token usage, and whether the quality validation passed. Reads from the checkpoint file written by `kova build` or `kova team-build`.

## Usage

```bash
kova status [flags]
```

## Flags

| Flag            | Description                                                           | Default |
| --------------- | --------------------------------------------------------------------- | ------- |
| `--verbose`     | Show full task details including agent output for completed tasks     | false   |
| `--json`        | Output status as a JSON object for scripting                          | false   |
| `--plan <path>` | Show status for a specific plan checkpoint instead of the most recent | none    |

## Task States

| State     | Meaning                                                             |
| --------- | ------------------------------------------------------------------- |
| `pending` | Waiting for one or more dependencies to complete                    |
| `running` | Agent is actively executing                                         |
| `done`    | Completed successfully                                              |
| `failed`  | Agent returned an error                                             |
| `blocked` | Skipped because a dependency failed                                 |
| `skipped` | Skipped explicitly (e.g., `--no-validate` skips quality-validation) |

When a task has status `blocked`, the `waiting` column shows which dependency failed.

## Token Usage Display

The status output always shows session budget consumption:

```
Token usage: 58,300 used  |  Session budget: 66% of 88,000 window
```

Configure your Claude plan in `kova.yaml` under `usage_tracking.plan` (`pro`, `max5`, `max20`, or `api`) so Kova can calculate the correct window allocation. If token usage approaches `warn_at_percent` (default 80%), a warning appears:

```
WARNING: Session budget at 83% (73,040 / 88,000 tokens used)
Remaining tasks may exceed your 5-hour window.
```

## How It Works

`kova status` reads the most recent `.progress.json` checkpoint file in `.claude/tasks/`. It does not call Claude Code or dispatch any agents -- it is a pure read of the checkpoint on disk. If no checkpoint exists, it tells you there is no build in progress.
