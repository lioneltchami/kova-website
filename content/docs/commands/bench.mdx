# kova bench

Benchmark a prompt against multiple models side-by-side. Runs the same prompt through each model sequentially, records cost and response time, then displays a comparison table showing cost, speed, token usage, and response previews. By default, benchmarks the three routing tiers (haiku, sonnet, opus).

## Usage

```bash
kova bench <prompt> [flags]
```

## Flags

| Flag         | Type    | Default             | Description                                                   |
| ------------ | ------- | ------------------- | ------------------------------------------------------------- |
| `--models`   | string  | `haiku,sonnet,opus` | Comma-separated model names (e.g. `sonnet,gpt-4o,gemini-pro`) |
| `--no-tools` | boolean | false               | Disable tools for pure text comparison                        |

## Model Short Names

You can use short names instead of full model identifiers:

| Short Name     | Model            |
| -------------- | ---------------- |
| `haiku`        | claude-haiku-3-5 |
| `sonnet`       | claude-sonnet-4  |
| `opus`         | claude-opus-4    |
| `gpt-4o`       | gpt-4o           |
| `gpt-4.1`      | gpt-4.1          |
| `o3`           | o3               |
| `gemini-flash` | gemini-2.5-flash |
| `gemini-pro`   | gemini-2.5-pro   |

### Pure text comparison without tools

```bash
kova bench "summarize the pros and cons of microservices" --no-tools
```

Disabling tools ensures all models produce text-only responses, making the comparison more apples-to-apples.
