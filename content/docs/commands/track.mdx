# kova track

Scan local AI tool data directories and write usage records to `~/.kova/usage.json`. Running `kova track` without flags scans all enabled collectors and processes any new events since the last scan.

You do not need to pass token counts manually -- Kova reads them from each tool's own data files. Use the manual flags only if you need to backfill an event from a tool that has no native collector.

## Usage

```bash
kova track [flags]
```

## Flags

| Flag              | Type    | Default | Description                                                              |
| ----------------- | ------- | ------- | ------------------------------------------------------------------------ |
| `--tool`          | string  | all     | Only scan this tool: `claude_code`, `cursor`, `copilot`, `windsurf`, `devin`, `aider`, `continue_dev`, `cline`, `amazon_q`, `bolt`, `lovable` |
| `--daemon`        | boolean | false   | Run continuously in the background, scanning on a regular interval       |
| `--auto-sync`     | boolean | false   | After each scan, automatically run `kova sync` to push records to the dashboard |
| `--since`         | string  | --      | Only collect events after this date (ISO 8601 or relative, e.g. `7d`)  |
| `--model`         | string  | --      | (Manual) Model name for a single event (e.g. `claude-sonnet-4-5`)       |
| `--input-tokens`  | number  | --      | (Manual) Input/prompt token count for a single event                    |
| `--output-tokens` | number  | --      | (Manual) Output/completion token count for a single event               |
| `--cost`          | number  | --      | (Manual) Total cost in USD for a single event                           |
| `--project`       | string  | --      | (Manual) Project name for cost grouping                                 |
| `--session-id`    | string  | --      | (Manual) Session identifier for grouping related events                 |
| `--duration-ms`   | number  | --      | (Manual) Duration of the API call in milliseconds                       |
| `--at`            | string  | now     | (Manual) Timestamp for a single event (ISO 8601)                        |

## Examples

### Scan All Tools

```bash
kova track
```

```
Scanning collectors...
  claude_code    12 new events   $4.82
  cursor          5 new events   $1.20
  copilot         0 new events
  windsurf        2 new events   $0.31
  aider           3 new events   $0.14
  continue_dev    1 new event    $0.07

Recorded 23 events. Run `kova sync` to upload.
```

### Scan a Single Tool

```bash
kova track --tool claude_code
```

### Run as a Background Daemon with Auto-Sync

Keep the dashboard current automatically without any manual steps:

```bash
kova track --daemon --auto-sync
```

The daemon scans all enabled collectors on a regular interval (default: every 15 minutes) and syncs new records to the cloud after each pass. Terminate with `Ctrl+C`.

For a system-level setup that survives reboots, see the [Automatic Sync guide](/docs/guides/auto-sync).

### Manual Event (Backfill)

Use the manual flags to add an event from a tool that has no native collector:

```bash
kova track \
  --tool cursor \
  --model gpt-4o \
  --input-tokens 800 \
  --output-tokens 250 \
  --cost 0.003
```

Backfill a historical event with a specific timestamp:

```bash
kova track \
  --tool copilot \
  --model gpt-4 \
  --input-tokens 1200 \
  --output-tokens 400 \
  --cost 0.006 \
  --at "2026-03-15T14:30:00Z"
```

## How Events Are Stored

Events are appended to `~/.kova/usage.json` as a flat JSON array. Each event gets a unique ID derived from a hash of the tool, timestamp, and token counts, making uploads idempotent -- running `kova sync` multiple times will not create duplicate records in the cloud.
