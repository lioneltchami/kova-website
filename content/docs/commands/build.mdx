# kova build

Execute a plan file using hub-and-spoke coordination. The orchestrator dispatches specialist agents in dependency order, running independent tasks in parallel, collecting results, and propagating failures to dependent tasks. Progress is checkpointed after each task so interrupted builds can resume without repeating completed work.

## Usage

```bash
kova build [plan-path] [flags]
```

If `plan-path` is omitted, Kova uses the most recently created plan in `.claude/tasks/`.

## Flags

| Flag                       | Description                                             | Default          |
| -------------------------- | ------------------------------------------------------- | ---------------- |
| `--resume`                 | Resume a previously interrupted build from checkpoint   | false            |
| `--parallel <n>`           | Maximum concurrent agent tasks                          | From `kova.yaml` |
| `--model-override <model>` | Override model for all tasks: `haiku`, `sonnet`, `opus` | none             |
| `--dry-run`                | Show execution plan without running agents              | false            |
| `--verbose`                | Show detailed agent output for each task                | false            |
| `--no-validate`            | Skip quality-engineer validation step                   | false            |
| `--live`                   | Show live streaming output from agents as they execute  | false            |

## Examples

### Basic Build

Uses the most recent plan in `.claude/tasks/`:

```bash
kova build
```

### Build a Specific Plan

```bash
kova build .claude/tasks/add-user-auth.md
```

### Resume After Interruption

Reads the checkpoint file and skips tasks that already completed:

```bash
kova build --resume
```

Sample resume output:

```

  [skip] build-login-form       done in previous run (1m 55s)

  [running] wire-auth-provider   sonnet   starting...
```

### Dry Run Preview

Shows the execution plan without dispatching any agents:

```bash
kova build --dry-run
```

Output:


### Override All Task Models

Force all tasks to use `haiku` to minimize cost (lower quality):

```bash
kova build --model-override haiku
```

### Live Streaming Output

Show each agent's output as it streams in real-time:

```bash
kova build --live
```

## How It Works

1. **Parse Plan**: Read the plan markdown file and extract tasks, agents, models, and `depends_on` relationships.
2. **Build Graph**: Create a directed acyclic graph of task dependencies.
3. **Identify Roots**: Find all tasks with no dependencies (Phase 1 candidates).
4. **Dispatch**: Run Phase 1 tasks in parallel up to `max_parallel_agents`.
5. **Progress**: As tasks complete, unlock and dispatch their dependents.
6. **Checkpoint**: After each task completes, write progress to `.claude/tasks/<plan>.progress.json`.
7. **Validate**: After all tasks complete, dispatch `quality-engineer` unless `--no-validate` is set.
8. **Report**: Print the build summary with timing, token usage, and cost estimate.

## Failure Handling

When a task fails:

- Kova marks it `failed` in the checkpoint file
- Tasks that depend on the failed task are set to `blocked` and skipped
- Tasks with no dependency on the failed task continue to run
- The build completes with status `PARTIAL` or `FAILED`

To retry after a failure, fix the underlying issue (usually an agent error or file conflict) then run:

```bash
kova build --resume
```

The resume will skip `completed` tasks and retry `failed` and `blocked` tasks from the beginning.

## Checkpoint File

After every task, Kova writes a checkpoint to `.claude/tasks/<plan-slug>.progress.json`:

```json
{
  "plan": "add-user-auth.md",
  "started_at": "2026-03-14T10:00:00Z",
  "status": "in_progress",
  "tasks": {
    "create-auth-schema": {
      "status": "completed",
      "agent_type": "supabase-specialist",
      "model": "opus",
      "started_at": "2026-03-14T10:00:05Z",
      "completed_at": "2026-03-14T10:00:52Z",
      "duration_s": 47,
      "tokens": { "input": 12400, "output": 3100, "total": 15500 }
    },
    "build-auth-api": {
      "status": "in_progress",
      "agent_type": "backend-engineer",
      "model": "sonnet",
      "started_at": "2026-03-14T10:00:53Z"
    }
  }
}
```
