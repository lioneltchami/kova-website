# kova logout

Delete the locally stored Kova credentials. After logging out, commands that require authentication (`kova sync`, `kova costs` with cloud data) will stop working until you run `kova login` again.

Logging out does not invalidate your API key server-side. The key remains active and can be used again with `kova login`. To revoke a key entirely, go to **Dashboard > Settings > API Keys**.

## Usage

```bash
kova logout
```

## Examples

### Log Out

```bash
kova logout
```

```
Logged out successfully.
Your local credentials have been removed.
```

### Already Logged Out

If no credentials are stored, `kova logout` exits cleanly:

```bash
kova logout
```

```
Not logged in.
```

## What Gets Removed

`kova logout` removes `~/.kova/credentials.json`. Local usage records stored in `~/.kova/usage/` are not affected -- your data remains on disk and can be synced after you log in again.
