# kova config

View and edit your Kova configuration without opening the file manually. Supports reading the full config, setting individual values by dot-notation key path, and managing API keys.

## Usage

```bash
kova config [subcommand] [args]
```

Running `kova config` without a subcommand displays the current configuration.

## Subcommands

| Subcommand                      | Description                                   |
| ------------------------------- | --------------------------------------------- |
| `kova config`                   | Display current configuration as YAML         |
| `kova config set <key> <value>` | Set a configuration value by dot-notation key |
| `kova config get <key>`         | Print the value of a single key               |

## Examples

### View Configuration

```bash
kova config
```

Output:

```yaml
version: 1

auth:
  api_key: kova_xxxx...

collectors:
  claude_code:
    enabled: true
  cursor:
    enabled: true
    db_path: null
  copilot:
    enabled: false
    monthly_cost: 19.00
  windsurf:
    enabled: false
    api_key: null
  devin:
    enabled: false
    api_key: null

sync:
  auto_sync: false
  upload_since_days: 7

display:
  currency: USD
  timezone: local
```

### Enable a Collector

```bash
kova config set collectors.cursor.enabled true
kova config set collectors.windsurf.enabled true
kova config set collectors.windsurf.api_key "cwk_your_api_key"
```

### Configure Copilot Flat-Rate Cost

```bash
kova config set collectors.copilot.enabled true
kova config set collectors.copilot.monthly_cost 19.00
```

### Set Notification Webhooks

```bash
kova config set notifications.budget_warn.slack "https://hooks.slack.com/services/TEAM/CHANNEL/TOKEN"
kova config set notifications.budget_warn.discord "https://discord.com/api/webhooks/ID/TOKEN"
```

### Change Sync Defaults

```bash
kova config set sync.upload_since_days 30
kova config set sync.auto_sync true
```

### Change Timezone for Cost Grouping

```bash
kova config set display.timezone "America/New_York"
```

### Get a Single Value

```bash
kova config get collectors.cursor.enabled
# true
```

## Key Reference

Common dot-notation keys for `kova config set`:

| Key                               | Type    | Example                       |
| --------------------------------- | ------- | ----------------------------- |
| `collectors.claude_code.enabled`  | boolean | `true`                        |
| `collectors.cursor.enabled`       | boolean | `true`                        |
| `collectors.cursor.db_path`       | string  | `/custom/path/to/cursor/data` |
| `collectors.copilot.enabled`      | boolean | `true`                        |
| `collectors.copilot.monthly_cost` | number  | `19.00`                       |
| `collectors.windsurf.enabled`     | boolean | `true`                        |
| `collectors.windsurf.api_key`     | string  | `cwk_...`                     |
| `collectors.devin.enabled`        | boolean | `true`                        |
| `collectors.devin.api_key`        | string  | `dv_...`                      |
| `sync.auto_sync`                  | boolean | `false`                       |
| `sync.upload_since_days`          | number  | `30`                          |
| `notifications.budget_warn.slack` | string  | `https://hooks.slack.com/...` |
| `display.timezone`                | string  | `America/Chicago`             |
| `display.currency`                | string  | `USD`                         |

## Configuration File Location

`kova config` reads and writes `~/.kova/config.yaml`. This is a global user config, not a per-project file. Settings apply to all projects on this machine.
