# kova init

Initialize Kova in your project. Detects your tech stack, scaffolds the `.claude/` directory with specialist agent definitions, skill files, and hook validators, and generates a `kova.yaml` configuration file.

## Usage

```bash
kova init [flags]
```

## Flags

| Flag              | Description                                                                     | Default |
| ----------------- | ------------------------------------------------------------------------------- | ------- |
| `--force`         | Reinitialize even if `kova.yaml` already exists                                 | false   |
| `--merge`         | Merge new templates into existing `.claude/` without overwriting customizations | false   |
| `--dry-run`       | Show what would be created without writing any files                            | false   |
| `--no-detect`     | Skip auto-detection, use framework-agnostic defaults                            | false   |
| `--preset <name>` | Use a named preset: `nextjs`, `expo`, `django`, `express`, `react`, `node`      | none    |

## Examples

### Basic Initialization

```bash
cd your-project
kova init
```

### Force Reinitialize

Wipes and recreates `.claude/` and `kova.yaml` from scratch:

```bash
kova init --force
```

### Merge Into Existing Setup

Adds missing templates without overwriting your customized skill files:

```bash
kova init --merge
```

### Preview Without Writing

Shows the full list of files that would be created:

```bash
kova init --dry-run
```

### Use a Framework Preset

Starts with opinionated defaults for a specific framework:

```bash
kova init --preset nextjs
kova init --preset expo
kova init --preset django
```

## Interactive Mode

When run in a terminal without flags, `kova init` enters interactive mode and walks through configuration step by step. Auto-detected values are shown as defaults -- press Enter to accept or type a new value.

```
? Project name: my-app
? Project description: TypeScript Next.js + Supabase application
? Primary language: typescript (detected)
? Framework: next (detected)
? Default model tier: sonnet
? Max parallel agents: 4
? Protected files (comma-separated globs): .env*, secrets/**, *.lock
```

Pass any flag to skip interactive mode and use detected values directly.

## Auto-Detection

Kova examines your project directory and infers configuration from the files it finds:

| Signal                                       | What Gets Detected   |
| -------------------------------------------- | -------------------- |
| `tsconfig.json`                              | TypeScript language  |
| `package.json` deps: `next`                  | Next.js framework    |
| `package.json` deps: `expo`                  | Expo framework       |
| `package.json` deps: `react`                 | React framework      |
| `package.json` deps: `express`               | Express framework    |
| `package.json` deps: `@supabase/supabase-js` | Supabase database    |
| `package.json` deps: `prisma`                | Prisma database      |
| `package.json` deps: `stripe`                | Stripe payments      |
| `package.json` deps: `better-auth`           | BetterAuth           |
| `package-lock.json`                          | npm package manager  |
| `pnpm-lock.yaml`                             | pnpm package manager |
| `yarn.lock`                                  | Yarn package manager |
| `bun.lockb`                                  | Bun package manager  |
| `scripts.test` in package.json               | Test command         |
| `scripts.lint` in package.json               | Lint command         |
| `pyproject.toml`                             | Python language      |
| `go.mod`                                     | Go language          |
| `Cargo.toml`                                 | Rust language        |

Detected values populate `kova.yaml` and the `CLAUDE.md` project instructions file.
