# Quick Start

This guide walks through the full Kova workflow: initialize your project, plan a task, execute the build, and open a pull request. The entire sequence takes about 5 minutes for a real feature.

## Step 1: Initialize

Run `kova init` in your project root. Kova detects your tech stack automatically and scaffolds the `.claude/` directory with specialist agent definitions, skill files, and hook validators.

```bash
cd your-project
kova init
```

Expected output:

```
Detected:
  Language:    TypeScript
  Framework:   Next.js 15
  Package Mgr: pnpm
  Database:    Supabase
  Auth:        BetterAuth
  Test:        vitest
  Lint:        biome check

Created:
  .claude/commands/team-plan.md
  .claude/commands/build.md
  .claude/commands/team-build.md
  .claude/skills/session-management/SKILL.md
  .claude/skills/sub-agent-invocation/SKILL.md

Next steps:
  1. Review kova.yaml and adjust settings
  2. Run: kova plan "your task description"
  3. Review the plan, then: kova build
```

Review the generated `kova.yaml` and adjust boundaries or model settings if needed. See [Configuration](/docs/getting-started/configuration) for details.

## Step 2: Plan

Describe what you want to build. Kova analyzes your codebase, maps dependencies, designs a phased approach, and produces a structured task graph before writing a single line of code.

```bash
kova plan "add stripe payment processing to the checkout flow"
```

Expected output:

```
Planning...  (using opus model)

Analyzing codebase...
  Found 34 files in src/
  Identified app/checkout.tsx (current checkout page)
  Detected Stripe publishable key already configured
  Found existing stripe-checkout edge function

Plan created: .claude/tasks/stripe-payment-checkout.md

Summary:
  Objective:  Add Stripe payment processing to checkout flow
  Phases:     3 (API updates, components, validation)
  Tasks:      7
  Agents:     frontend-specialist, backend-engineer, quality-engineer
  Est. Models: 2x haiku, 3x sonnet, 1x opus, 1x sonnet (validation)

Task Graph:
  1. [haiku]  update-api-contract          (no deps)
  2. [sonnet] implement-payment-intent     (depends on 1)
  3. [sonnet] build-payment-form           (depends on 1)
  4. [sonnet] wire-stripe-elements         (depends on 2, 3)
  5. [haiku]  update-confirmation-screen   (depends on 4)
  6. [sonnet] integration-tests            (depends on 4, 5)
  7. [sonnet] quality-validation           (depends on all)

Proceed with build? [Y/n/edit]
```

Open the generated plan file at `.claude/tasks/stripe-payment-checkout.md` to review task descriptions, acceptance criteria, and agent assignments before proceeding. Press `Y` to confirm.

## Step 3: Build

Execute the plan. Kova dispatches specialist agents in dependency order, runs them in parallel where possible, and streams live progress to your terminal.

```bash
kova build --live
```

Expected output:

```
Building: stripe-payment-checkout.md
Progress: [=========>      ] 5/7 tasks (71%)

  [done]    update-api-contract          haiku    38s
  [done]    implement-payment-intent     sonnet   2m 15s
  [done]    build-payment-form           sonnet   1m 48s
  [done]    wire-stripe-elements         sonnet   2m 02s
  [done]    update-confirmation-screen   haiku    42s
  [running] integration-tests            sonnet   started 1m ago
  [pending] quality-validation           sonnet

Estimated remaining: ~3 minutes
```

When the build finishes, Kova prints a summary showing task results, token usage, and estimated cost:

```
Build Complete: stripe-payment-checkout.md

  Tasks: 7/7 passed
  Duration: 8m 47s
  Token Usage: 105,330 total
  Cost Estimate: $0.34
  Savings from model tiering: $0.82 (vs all-sonnet)
```

## Step 4: Create a Pull Request

Once the build passes, open a pull request directly from the CLI. Kova generates a title and body from the plan objective and completed task list.

```bash
kova pr
```

Expected output:

```
Creating PR...

Title: feat: add stripe payment processing to checkout flow

PR created: https://github.com/your-org/your-repo/pull/47
```

## The 30-Second Workflow

Once you are familiar with the commands, the day-to-day workflow is:

```bash
kova plan "your task"   # Design the approach
kova build              # Execute with specialist agents
kova pr                 # Open the pull request
```

Or collapse planning and building into one step:

```bash
kova run "your task"
```
