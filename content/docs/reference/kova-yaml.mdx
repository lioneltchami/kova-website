# Configuration Reference

Kova stores all local configuration in `~/.kova/config.json`. The file is created automatically on first run of `kova init` or `kova track`. Edit it directly or use `kova config` and `kova budget` commands to update values.

## File Location

```
~/.kova/config.json
```

## Complete Default Configuration

```json
{
  "budget": {
    "monthly_usd": null,
    "daily_usd": null,
    "warn_at_percent": 80
  },
  "collectors": {},
  "auto_sync": false,
  "sso": null
}
```

---

## budget

Controls spending limits and alert thresholds. When a budget is set, `kova costs` and `kova track` show your current spend vs. the limit.

| Field             | Type           | Default | Description                                                                        |
| ----------------- | -------------- | ------- | ---------------------------------------------------------------------------------- |
| `monthly_usd`     | number or null | `null`  | Monthly spending limit in USD. Null means no limit is enforced.                    |
| `daily_usd`       | number or null | `null`  | Daily spending limit in USD. Null means no limit is enforced.                      |
| `warn_at_percent` | number         | `80`    | Percentage of budget at which a warning is shown. Default is 80 (warn at 80% used). |

**Set via CLI:**

```bash
kova budget --monthly 150
kova budget --daily 10
kova budget --warn-at 70
```

**Example:**

```json
{
  "budget": {
    "monthly_usd": 150,
    "daily_usd": 10,
    "warn_at_percent": 70
  }
}
```

## collectors

Per-collector configuration, primarily used to store API keys. Keys are stored under the tool's collector key name.

| Collector key  | Description                                                     |
| -------------- | --------------------------------------------------------------- |
| `cursor`       | Cursor API key for fetching usage events                        |
| `windsurf`     | Windsurf service key                                            |
| `devin`        | Devin Enterprise API key                                        |
| `copilot`      | GitHub Personal Access Token (PAT) with `read:billing` scope    |
| `amazon_q`     | AWS credentials in `ACCESS_KEY_ID:SECRET_ACCESS_KEY` format     |
| `bolt`         | Bolt.new API key (stored for future use when API is available)  |
| `lovable`      | Lovable API key (stored for future use when API is available)   |

**Set via CLI:**

```bash
kova config set-key cursor <your-cursor-api-key>
kova config set-key windsurf <your-windsurf-service-key>
kova config set-key devin <your-devin-api-key>
kova config set-key copilot <github-pat>
kova config set-key amazon_q <ACCESS_KEY_ID>:<SECRET_ACCESS_KEY>
```

**Example:**

```json
{
  "collectors": {
    "cursor": "cursor_sk_abc123",
    "devin": "devin_api_xyz789"
  }
}
```

API keys are stored in plain text in `~/.kova/config.json`. The file is created with mode `0600` (owner read/write only). Do not commit this file to version control.

## auto_sync

When `true`, Kova automatically uploads usage records to the cloud dashboard after each `kova track` run. Requires an active login (`kova login`).

| Field       | Type    | Default | Description                                                          |
| ----------- | ------- | ------- | -------------------------------------------------------------------- |
| `auto_sync` | boolean | `false` | Enable automatic sync to the cloud dashboard after each track scan.  |

**Set via CLI:**

```bash
kova config set auto_sync true
```

**Example:**

```json
{
  "auto_sync": true
}
```

## sso

SSO configuration set by `kova sso configure`. Null when SSO is not configured.

| Field              | Type           | Description                                                          |
| ------------------ | -------------- | -------------------------------------------------------------------- |
| `enabled`          | boolean        | True when SSO is configured and active.                              |
| `issuer`           | string or null | OIDC issuer URL, e.g. `https://sso.mycompany.com`.                   |
| `token`            | string or null | Stored OAuth token. Set automatically after `kova sso login`.        |
| `token_expires_at` | string or null | ISO 8601 expiry time of the stored token. Null if unknown.           |

**Set via CLI:**

```bash
kova sso configure --issuer https://sso.mycompany.com
kova sso login
```

**Example:**

```json
{
  "sso": {
    "enabled": true,
    "issuer": "https://sso.mycompany.com",
    "token": "eyJhbGci...",
    "token_expires_at": "2026-03-20T09:00:00.000Z"
  }
}
```

## cost_centers

Cost center mappings created by `kova tag`. Each cost center has an ID, name, and a list of project names that belong to it.

| Field      | Type     | Description                                      |
| ---------- | -------- | ------------------------------------------------ |
| `id`       | string   | URL-safe kebab-case identifier, auto-generated.  |
| `name`     | string   | Display name for the cost center.                |
| `projects` | string[] | Project names mapped to this cost center.        |

**Set via CLI:**

```bash
kova tag my-project --cost-center Engineering
```

**Example:**

```json
{
  "cost_centers": [
    {
      "id": "engineering",
      "name": "Engineering",
      "projects": ["kova-api", "kova-website", "kova-cli"]
    },
    {
      "id": "data-platform",
      "name": "Data Platform",
      "projects": ["etl-pipeline", "analytics-service"]
    }
  ]
}
```
