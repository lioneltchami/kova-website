# Usage Data Format

Kova writes all collected usage data to `~/.kova/usage.json`. This file is the source of truth for `kova costs`, `kova report`, `kova compare`, and `kova sync`. Understanding its format lets you write scripts against it, troubleshoot missing data, or import records from other sources.

## File Location

```
~/.kova/usage.json
```

The file is created on first run of `kova track` or `kova init`. It is plain JSON and can be opened in any text editor.

## Top-Level Structure

```json
{
  "records": [...],
  "last_scan": "2026-03-19T14:00:00.000Z"
}
```

| Field       | Type            | Description                                                      |
| ----------- | --------------- | ---------------------------------------------------------------- |
| `records`   | UsageRecord[]   | Array of all collected usage records, newest last.               |
| `last_scan` | ISO 8601 string | Timestamp of the most recent collector scan. Null if never run.  |

## UsageRecord Fields

Each element in the `records` array is a `UsageRecord` object:

| Field           | Type               | Description                                                                           |
| --------------- | ------------------ | ------------------------------------------------------------------------------------- |
| `id`            | string             | Unique record ID. Deterministic hash of the source data. Prevents duplicate imports.  |
| `tool`          | string             | Source AI tool. See [tool values](#tool-values) below.                                |
| `model`         | string             | Model used. See [model values](#model-values) below.                                  |
| `session_id`    | string             | Conversation or session identifier from the source tool.                              |
| `project`       | string or null     | Project directory name detected from the working directory. Null if not available.    |
| `input_tokens`  | number             | Input (prompt) token count for this record.                                           |
| `output_tokens` | number             | Output (completion) token count for this record.                                      |
| `cost_usd`      | number             | Estimated cost in USD. Calculated from token counts and model rates.                  |
| `timestamp`     | ISO 8601 string    | When this usage occurred, as reported by the source tool.                             |
| `duration_ms`   | number or null     | Session or request duration in milliseconds. Null when not available.                 |
| `metadata`      | `Record<string, string>` | Tool-specific additional fields. Never contains credentials or API keys.          |

## Deduplication

Records have deterministic IDs computed from the source data (tool, session, timestamp, token counts). When `kova track` runs multiple times, duplicate records are detected by ID and not written twice. The same deduplication runs server-side during `kova sync`.
