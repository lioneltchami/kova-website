# Collectors Reference

Kova ships 11 collectors, one for each supported AI tool. Each collector knows where the tool stores its usage data on disk (or which API to call), how to parse the format, and how to convert raw entries into Kova's normalized `UsageRecord` schema.

Collectors run during `kova track` and `kova init`. You can target a single collector with `kova track --tool <name>`.

---

## Claude Code

**Collector key:** `claude_code`

**Source:** Local JSONL conversation history files

| Platform | Data path |
| -------- | --------- |
| macOS    | `~/.claude/projects/**/*.jsonl` |
| Linux    | `~/.claude/projects/**/*.jsonl` |
| Windows  | `%USERPROFILE%\.claude\projects\**\*.jsonl` |

The path root can be overridden with the `CLAUDE_HOME` environment variable.

**Data collected:** input tokens, output tokens (including cache creation and cache read tokens), model name (haiku/sonnet/opus), session ID, project directory, timestamp, stop reason.

**Cost calculation:** Token costs are applied using `TOKEN_COSTS` rates per model. Cache read tokens are billed at a reduced rate.

## Cursor

**Collector key:** `cursor`

**Source:** Cursor API (requires API key stored via `kova config set-key cursor <key>`)

| Platform | Local state database (used for availability check) |
| -------- | -------------------------------------------------- |
| macOS    | `~/Library/Application Support/Cursor/User/globalStorage/state.vscdb` |
| Linux    | `~/.config/Cursor/User/globalStorage/state.vscdb` |
| Windows  | `%APPDATA%\Cursor\User\globalStorage\state.vscdb` |

**Data collected:** timestamp, model, input tokens, output tokens, cache read/write tokens, charged cents per event.

**Cost calculation:** Uses Cursor pool rates: input $1.25/1M, output $6.00/1M, cache read $0.25/1M, cache write $1.25/1M. If `chargedCents` is present on the event, that value is used directly.

## GitHub Copilot

**Collector key:** `copilot`

**Source:** Local VS Code extension chat session files, with optional GitHub Billing API fallback (requires PAT stored via `kova config set-key copilot <pat>`)

| Platform | Chat session path |
| -------- | ----------------- |
| macOS    | `~/Library/Application Support/Code/User/globalStorage/github.copilot-chat/` |
| Linux    | `~/.config/Code/User/globalStorage/github.copilot-chat/` |
| Windows  | `%APPDATA%\Code\User\globalStorage\github.copilot-chat\` |

**Data collected:** conversation ID, model, prompt token count, completion token count, timestamp per request.

**Cost calculation:** Token costs applied using `TOKEN_COSTS` rates. Copilot is a flat-rate subscription for most users, so `cost_usd` will be `0` unless token-level data is available from the billing API.

---

## Windsurf

**Collector key:** `windsurf`

**Source:** Windsurf API (requires service key stored via `kova config set-key windsurf <key>`)

**Data collected:** credit usage per session, model (SWE-1.5, SWE-1.5-fast), timestamps, session metadata.

**Cost calculation:** Credits are converted to USD using plan rates: Pro plan $0.02/credit, Teams plan $0.04/credit.

---

## Devin

**Collector key:** `devin`

**Source:** Devin Enterprise API at `https://api.devin.ai/v3/enterprise/consumption/daily` (requires API key stored via `kova config set-key devin <key>`)

**Data collected:** daily ACU (Agent Compute Unit) consumption, session count, date.

**Cost calculation:** ACUs converted to USD at $2.25/ACU (Core plan) or $2.00/ACU (Teams plan). The plan tier is inferred from account data returned by the API.

---

## Aider

**Collector key:** `aider`

**Source:** Local chat history files and cost report files

| File type | Locations searched |
| --------- | ------------------ |
| Chat history | `~/.aider.chat.history.md`, `<cwd>/.aider.chat.history.md` |
| Cost reports | `~/.aider/reports/`, `<cwd>/.aider/reports/` |

Aider writes both a markdown chat history and structured JSON cost reports when `--cost` is enabled.

**Data collected:** model name, input tokens, output tokens, cost per session (from Aider's own cost tracking when available), timestamp.

**Cost calculation:** Uses Aider's reported cost if present in the report file. Falls back to `TOKEN_COSTS` token rate lookup by model name.

---

## Continue.dev

**Collector key:** `continue_dev`

**Source:** Local session files written by the Continue VS Code extension

| Platform | Sessions directory |
| -------- | ------------------ |
| All      | `~/.continue/sessions/` |

**Data collected:** model, input/output tokens per prompt log entry, session ID, project, timestamp.

**Cost calculation:** Token costs applied per `TOKEN_COSTS` rates. Continue.dev supports many model providers; the collector normalizes model names from the raw `completionOptions.model` or `modelTitle` field.

---

## Cline

**Collector key:** `cline`

**Source:** VS Code extension globalStorage task files (one JSON file per Cline task)

| Platform | Storage path |
| -------- | ------------ |
| macOS    | `~/Library/Application Support/Code/User/globalStorage/saoudrizwan.claude-dev/` |
| Linux    | `~/.config/Code/User/globalStorage/saoudrizwan.claude-dev/` |
| Windows  | `%APPDATA%\Code\User\globalStorage\saoudrizwan.claude-dev\` |

**Data collected:** task ID, model, input/output tokens, cost estimate (from Cline's own tracking), timestamp.

**Cost calculation:** Uses Cline's recorded cost per task when available. Falls back to `TOKEN_COSTS` rate lookup.

---

## Amazon Q Developer

**Collector key:** `amazon_q`

**Source:** AWS Cost Explorer API. Requires AWS credentials stored via `kova config set-key amazon_q <ACCESS_KEY_ID>:<SECRET_ACCESS_KEY>`, or ambient AWS credentials (environment variables, `~/.aws/credentials`, or EC2 instance role).

**Data collected:** daily token usage and cost from the AWS billing API, filtered to Amazon Q Developer service entries.

**Cost calculation:** Uses Cost Explorer's reported cost directly. Falls back to an estimated rate of $3.00/1M input tokens and $15.00/1M output tokens when token-level billing data is unavailable.

---

## Bolt.new

**Collector key:** `bolt`

**Source:** Bolt.new API (pending public availability). Requires API key stored via `kova config set-key bolt <key>`.

**Data collected:** session usage and cost as reported by the Bolt.new API.

**Note:** Bolt.new does not currently expose a public usage API. When the API becomes available, Kova will collect usage automatically. In the meantime, the collector stores your API key and returns empty records.

---

## Lovable

**Collector key:** `lovable`

**Source:** Lovable API (pending public availability). Requires API key stored via `kova config set-key lovable <key>`.

**Data collected:** session usage and cost as reported by the Lovable API.

**Note:** Similar to Bolt.new, Lovable does not yet expose a public usage API. The collector will activate automatically when the API is available.

## Collector Availability Check

Each collector implements an `isAvailable()` method that Kova calls during `kova init` and `kova track` to determine whether the tool is installed. The check varies by collector:

- **File-based collectors** (Claude Code, Copilot, Cline, Continue.dev): check whether the data directory exists on disk.
- **API-based collectors** (Cursor, Windsurf, Devin, Amazon Q, Bolt, Lovable): check whether an API key is stored in the Kova credential store.
- **Aider**: checks whether any `.aider.chat.history.md` file exists in the home directory or current working directory.
