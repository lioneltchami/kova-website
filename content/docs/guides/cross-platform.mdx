# Cross-Platform

Kova runs on Windows, macOS, and Linux. The CLI interface and config format are identical across platforms. The differences are in file path handling, atomic write implementation, and where each AI tool stores its local data files. This page documents those differences.

## Platform Detection

Kova reads `process.platform` at runtime to select platform-specific code paths:

```text
"win32"  -- Windows
"darwin" -- macOS
"linux"  -- Linux
```

Platform detection is automatic. You do not configure it.

---

## Collector Data Paths by Platform

Each AI tool stores its local usage data in a platform-specific location. Kova auto-detects these paths on macOS and Linux. On Windows, some paths require manual configuration if auto-detection fails.

| Tool         | macOS                                                                           | Linux                                                       | Windows                                                     |
| ------------ | ------------------------------------------------------------------------------- | ----------------------------------------------------------- | ----------------------------------------------------------- |
| Claude Code  | `~/.claude/`                                                                    | `~/.claude/`                                                | `%APPDATA%\Claude\`                                         |
| Cursor       | `~/Library/Application Support/Cursor/User/globalStorage/`                      | `~/.config/Cursor/User/globalStorage/`                      | `%APPDATA%\Cursor\User\globalStorage\`                      |
| Copilot      | `~/Library/Application Support/GitHub Copilot/`                                 | `~/.config/github-copilot/`                                 | `%APPDATA%\GitHub Copilot\`                                 |
| Windsurf     | API-based (no local files)                                                      | API-based (no local files)                                  | API-based (no local files)                                  |
| Aider        | `~/.aider/`                                                                     | `~/.aider/`                                                 | `%USERPROFILE%\.aider\`                                     |
| Continue.dev | `~/.continue/`                                                                  | `~/.continue/`                                              | `%USERPROFILE%\.continue\`                                  |
| Cline        | `~/Library/Application Support/Code/User/globalStorage/saoudrizwan.claude-dev/` | `~/.config/Code/User/globalStorage/saoudrizwan.claude-dev/` | `%APPDATA%\Code\User\globalStorage\saoudrizwan.claude-dev\` |
| Amazon Q     | `~/.aws/amazonq/`                                                               | `~/.aws/amazonq/`                                           | `%USERPROFILE%\.aws\amazonq\`                               |
| Devin        | API-based (no local files)                                                      | API-based (no local files)                                  | API-based (no local files)                                  |
| Bolt         | API-based (no local files)                                                      | API-based (no local files)                                  | API-based (no local files)                                  |
| Lovable      | API-based (no local files)                                                      | API-based (no local files)                                  | API-based (no local files)                                  |

If Kova cannot find a tool's data at the auto-detected path, override it manually:

```bash
kova config set collectors.cursor.db_path "/custom/path/to/cursor/data"
```

## Windows

### Shell Execution

npm-installed CLI tools on Windows are `.cmd` wrappers. When Node.js spawns a `.cmd` file, it must use `shell: true` or the process fails with `ENOENT`. Kova sets `shell: true` automatically for all subprocesses on `win32`:

```typescript
// Kova internals -- simplified
const isWindows = process.platform === "win32";

await execa("claude", args, {
  cwd: projectDir,
  shell: isWindows, // true on Windows, false elsewhere
  stdio: "pipe",
});
```

This applies to all subprocess calls Kova makes. You do not need to configure anything.

### Path Separators

Windows uses backslashes (`\`) in filesystem paths. Kova normalizes all paths to forward slashes internally, including in config values and log output. You can use either separator when running commands.

### Atomic Checkpoint Writes

On Unix, atomic file writes use `rename()` (write to temp, rename to target). On Windows, Kova first unlinks the existing file, then renames the temp file to the target path:

```typescript
// Windows atomic write
await fs.unlink(targetPath).catch(() => {}); // ignore if not exists
await fs.rename(tempPath, targetPath);

// Unix atomic write
await fs.rename(tempPath, targetPath); // atomic on POSIX
```

This guarantees `~/.kova/usage.json` and `~/.kova/config.json` are never left in an inconsistent state.

### PowerShell and cmd

Kova works in both PowerShell and cmd without any difference in behavior:

```powershell
# PowerShell
kova track
kova sync
kova costs
kova ci-report
```

```cmd
:: cmd.exe
kova track
kova sync
kova costs
kova ci-report
```

### Fixing PATH Issues on Windows

After `npm install -g kova-cli`, if `kova` is not recognized:

```text
'kova' is not recognized as an internal or external command
```

The npm global bin directory is not in your PATH. Add it:

1. Find the path with: `npm config get prefix`
2. Add that path (plus `\` at the end) to your user PATH in System Properties > Environment Variables.

Alternatively, use `npx`:

```cmd
npx kova-cli track
npx kova-cli costs
```

Or restart your terminal after global install. Windows sometimes requires a new shell session for PATH changes to take effect.

## macOS

No special configuration is required on macOS. Install Node.js via Homebrew or nvm:

```bash
# Homebrew
brew install node
npm install -g kova-cli

# nvm
nvm install 22
nvm use 22
npm install -g kova-cli
```

Verify the install:

```bash
kova --version
kova track
```

### Shell Completions (macOS)

For zsh (default on macOS Catalina+):

```bash
kova completions zsh >> ~/.zshrc
source ~/.zshrc
```

For bash:

```bash
kova completions bash >> ~/.bash_profile
source ~/.bash_profile
```

## Linux

Standard Node.js installation works on all major Linux distributions.

### Install without sudo (recommended)

Use nvm to avoid needing sudo for global npm installs:

```bash
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Restart shell or source profile
source ~/.bashrc

# Install Node.js 22
nvm install 22
nvm use 22

# Install Kova
npm install -g kova-cli
```

### Shell Completions (Linux)

For bash:

```bash
kova completions bash >> ~/.bashrc
source ~/.bashrc
```

For zsh:

```bash
kova completions zsh >> ~/.zshrc
source ~/.zshrc
```

For fish:

```bash
kova completions fish > ~/.config/fish/completions/kova.fish
```

## CI/CD Environments

Kova works in GitHub Actions, GitLab CI, CircleCI, and other CI systems. All CI systems are treated as non-TTY environments, so `kova init` runs without prompts and uses defaults.

### GitHub Actions example

```yaml
name: Kova CI Report
on:
  push:
    branches: [main]

jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "22"

      - name: Install Kova
        run: npm install -g kova-cli

      - name: Sync and report
        env:
          KOVA_API_KEY: ${{ secrets.KOVA_API_KEY }}
        run: |
          kova sync
          kova ci-report
```

### Per-user config in CI

Kova reads config from `~/.kova/config.json`. In CI, set the API key via environment variable rather than committing credentials:

```bash
export KOVA_API_KEY=kova_xxxx...
kova sync
```

Do not commit `~/.kova/config.json`. Each developer and CI runner uses their own per-user config.
