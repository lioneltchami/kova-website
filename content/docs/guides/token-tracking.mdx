# Token Tracking

Kova collectors capture input and output token counts from each AI tool session. These token records power the cost calculations in `kova costs`, the analytics charts in the dashboard, and any custom queries you run with `--group-by`.

## How Collectors Capture Token Counts

Each collector reads token data from the tool's own local data files:

- **Claude Code**: Parses JSONL session files written by the Claude Code CLI. Each record includes `input_tokens`, `output_tokens`, `cache_read_input_tokens`, and `cache_creation_input_tokens`.
- **Cursor**: Reads from Cursor's local SQLite database, which records per-session token counts for API requests routed through Cursor's proxy.
- **Windsurf**: Polls the Windsurf usage API. Token counts are returned per Cascade session.
- **Copilot, Aider, Continue.dev, Cline, Amazon Q, Bolt, Lovable**: These tools use flat-rate or activity-based pricing. Where per-session token counts are not available, Kova records session duration and request counts and estimates token equivalents from known model characteristics.

A raw usage record in `~/.kova/usage.json` looks like this:

```json
{
  "id": "evt_abc123",
  "tool": "claude-code",
  "timestamp": "2026-03-19T14:00:00Z",
  "session_id": "sess_abc123",
  "model": "claude-sonnet-4-5",
  "input_tokens": 12000,
  "output_tokens": 3200,
  "cost_usd": 0.084
}
```

## Viewing Token Breakdowns with kova costs

The `kova costs` command displays token counts alongside cost figures in its default table output:

```bash
kova costs
```

```
AI Tool Costs -- March 2026
───────────────────────────────────────────────────────────
Tool            Cost        Sessions   Tokens (in/out)
claude-code     $42.18      94         1.2M / 384K
cursor          $18.40      312        820K / 210K
copilot         $5.17       --         flat-rate ($19/mo)
windsurf        $3.40       28         95K / 42K
───────────────────────────────────────────────────────────
Total           $69.15      434+
```

To see token counts grouped by model:

```bash
kova costs --group-by model --since this-month
```

To see token totals by day:

```bash
kova costs --group-by day --since 14d
```

To see token consumption for a single tool:

```bash
kova costs --tool claude-code --since 30d
```

## Token Cost Rates by Model

Kova uses the following per-model rates to calculate cost from token counts (approximate 2026 pricing):

| Model                 | Input per 1M tokens | Output per 1M tokens |
| --------------------- | ------------------- | -------------------- |
| claude-haiku-3-5      | $0.80               | $4.00                |
| claude-sonnet-4-5     | $3.00               | $15.00               |
| claude-opus-4         | $15.00              | $75.00               |
| gpt-4o                | $2.50               | $10.00               |
| gpt-4o-mini           | $0.15               | $0.60                |

Cost estimates are labeled approximate because exact pricing depends on your plan, regional pricing, cache hit rates, and any promotional pricing applied to your account. For the most current rates, see each provider's pricing page.

For tools with flat-rate subscriptions (Copilot, Bolt, Lovable), Kova amortizes your configured `monthly_cost` across active days rather than computing from token counts.

## Analyzing Token Consumption Patterns

The `--group-by` flag lets you slice token data along several dimensions:

```bash
# Which tool consumes the most tokens?
kova costs --group-by tool

# Which project drives the most token usage?
kova costs --group-by project --since this-month

# How does daily token consumption vary?
kova costs --group-by day --since 30d

# Which model accounts for the most spend?
kova costs --group-by model
```

Export the data for further analysis:

```bash
kova costs --format json --group-by model --since this-month > tokens-by-model.json
kova costs --format csv --group-by day --since 30d > daily-tokens.csv
```

## Token Data in the Dashboard

The Kova web dashboard at [kova.dev/dashboard](https://kova.dev/dashboard) surfaces token data in the Analytics section:

- **Token Trend**: Daily input and output token counts plotted over time, filterable by tool and project.
- **Model Breakdown**: Pie chart of token consumption and cost contribution by model.
- **Top Sessions**: Highest-token sessions ranked by total tokens, with drill-down to see the full session record.
- **Tool Comparison**: Side-by-side token and cost metrics for all enabled collectors.

Token data syncs to the dashboard whenever you run `kova sync`. The dashboard reflects your most recent sync.
